```typescript
  // Clear cache on state change
  if (wasWorking !== isWorking) {
    clearCacheOnStateChange(ctx.creep);
  }
  
  return isWorking;
}
```

## Why This Works

1. **Prevents Undefined State**: Always initializes `working` before using it
2. **Context-Aware**: Initialization based on current energy (not arbitrary default)
3. **Minimal Impact**: Only affects first evaluation, normal operation unchanged
4. **Defensive**: Guards against edge cases (respawn, manual memory clear, etc.)
5. **Intuitive**: "Has energy? Then work!" - natural bot behavior
